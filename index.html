Introduction

What is a Graph?

A collection of Vertices (nodes) and Edges (connections) that represent relationships between data points.

What is Traversal?

The process of visiting every node in a graph systematically exactly once.

Why do we need traversal?

  • Searching for specific information.
  • Finding the shortest path.
  • Analyzing connectivity in networks.
  • Detecting cycles or components.

What is BFS?

Definition

Breadth-First Search (BFS) is an algorithm for traversing or searching tree or graph data structures.

Key Idea: It starts at the tree root (or some arbitrary node of a graph) and explores all of the neighbor nodes at the present depth prior to moving on to the nodes at the next depth level.

Works on both Directed/Undirected Graphs and Trees.

Basic Concept

The "Ripple" Effect

Imagine dropping a stone into still water. The ripples expand outwards evenly in all directions.

BFS visits nodes in this exact pattern: Source → Neighbors → Neighbors of Neighbors.

Exploration Strategy

  1. Start from a source node.
  2. Visit all immediate neighbors (Distance 1).
  3. Then move to the next level nodes (Distance 2).
  4. Continue until all reachable nodes are visited.

Data Structure Used

QUEUE

(FIFO)

First In, First Out

Why a Queue?

BFS requires us to visit nodes in the order they were discovered. A Queue perfectly preserves this "chronological discovery" order.

When we discover neighbors of a node, we add them to the back of the queue. We always pick the next node to process from the front.

BFS Algorithm Steps

  1. Start with a source node.
  2. Mark the source as visited.
  3. Put the source node in a queue.
  4. While the queue is not empty:
    • Remove the front node from the queue.
    • Visit all its unvisited neighbors.
    • Mark them as visited and add them to the queue.

Pseudocode

BFS(Graph, start_node):

create queue Q

mark start_node as visited

enqueue start_node into Q

while Q is not empty:

node = dequeue Q

for each neighbor in Graph.neighbors(node):

if neighbor is NOT visited:

mark neighbor as visited

enqueue neighbor into Q

Example (Step-by-Step)

Sample Graph

Nodes: A, B, C, D, E

Edges: (A-B), (A-C), (B-D), (C-E)

Order of Traversal:

A → B → C → D → E

Steps:

  1. Enqueue A (Visited: A)
  2. Dequeue A, Enqueue neighbors B, C
  3. Dequeue B, Enqueue neighbor D
  4. Dequeue C, Enqueue neighbor E
  5. Dequeue D (No unvisited neighbors)
  6. Dequeue E (No unvisited neighbors)

Applications of BFS

Shortest Path

Finding the minimum number of edges between two nodes in an unweighted graph.

Social Networks

Friend suggestions and calculating "Degrees of Separation" (e.g., LinkedIn connections).

Web Crawling

Search engines use BFS to discover pages layer-by-layer starting from a seed URL.

GPS Navigation

Basic routing algorithms and finding nearby points of interest.

Time & Space Complexity

Time Complexity: $O(V + E)$

We visit every Vertex (V) once and check every Edge (E) exactly twice (once for each endpoint).

Space Complexity: $O(V)$

In the worst case (e.g., a star graph), the Queue might store all vertices at once.

In Simple Terms:

The algorithm's performance scales linearly with the size of the graph. It is very efficient for level-based analysis.

BFS vs DFS

Feature BFS DFS
Strategy Level-wise Depth-wise
Data Structure Queue Stack (or Recursion)
Memory More (stores levels) Less (stores path)
Best Use Case Shortest Path Pathfinding

Pros & Cons

Advantages

  • Always finds the shortest path in unweighted graphs.
  • Guaranteed to find a solution if one exists.
  • Simple logic and easy to implement.

Disadvantages

  • Requires more memory than DFS (storing all neighbors).
  • Not suitable for graphs with infinite or extremely deep branches.

Conclusion

Key Takeaways:

  • BFS is a layer-by-layer exploration algorithm.
  • It uses a Queue (FIFO) to manage discovery order.
  • Crucial for Shortest Path problems and connectivity.
  • Powers modern technology from Google Search to GPS.
